## Overview

A reusable GitHub Actions workflow template for deploying static sites via rsync over SSH. Born out of setting up the same deploy pipeline three times in a row and deciding to write it down properly once.

## What it does

1. Checks out the repo and runs your build command
2. Sets up an SSH key from secrets
3. rsyncs the build output to your server with `--delete` to remove stale files
4. Optionally runs a post-deploy script (cache flush, service restart, etc.)

## Usage

Drop the workflow file into `.github/workflows/` and set four secrets: `SSH_HOST`, `SSH_USER`, `SSH_KEY`, and `DEPLOY_PATH`. That's it.
